# 应用模式归纳概述 / Application Pattern Summary Overview

## 📚 概述 / Overview

**目标**: 归纳三大理论（Petri网、动态图论、拓扑模型）在典型领域的应用模式、常用分析流程与工具栈，形成可复用的决策参考。

**覆盖领域**:

- 操作系统（死锁预防、调度、锁依赖）
- 分布式系统（共识、复制、故障恢复）
- AI基础设施（计算图、数据管线、监控）

**输出**: 各领域的模式清单、决策树、示例案例、工具组合。

---

## 🧭 一、通用方法论

### 1.1 建模与选择

- 需求侧判定：是否需要形式化验证（→ Petri网），是否需要大规模时序追踪（→ 动态图），是否需要形状/拓扑特征（→ 拓扑模型）。
- 约束侧判定：实时性、规模、数据形态（事件流/日志/图/点云）。
- 产出物：模型选择决策树、输入数据规范、边界条件。

### 1.2 分析与验证

- Petri网：可达/活性/有界性；S/T不变量；展开与模型检验。
- 动态图：时间尊重路径、社区/中心性演化、谱漂移检测、增量/近似算法。
- 拓扑模型：持久同调、Hodge流分解、Mapper摘要。

### 1.3 工具链

- Petri网：CPN Tools / TLA+ / Spin / GreatSPN。
- 动态图：NetworkX/igraph、Flink/Kafka流、Neo4j、PyG/DGL。
- 拓扑：GUDHI/Ripser/giotto-tda、KeplerMapper。

## 🔧 二、领域概览

### 2.1 操作系统

- 核心问题：死锁、优先级反转、锁依赖、调度公平性。
- 推荐策略：Petri网建模+不变量/可达性；动态图跟踪锁依赖；拓扑检测循环/异常流。
- 产出物：`01-操作系统应用模式/` 决策树+案例（生产者-消费者、读写锁、锁图）。

### 2.2 分布式系统

- 核心问题：共识、复制一致性、分区容错、故障恢复。
- 推荐策略：Petri网验证协议安全性；动态图追踪副本拓扑/流量；拓扑分析韧性。
- 产出物：`02-分布式系统应用模式/` 决策树+案例（Raft日志、两阶段提交、反熵）。

### 2.3 AI基础设施

- 核心问题：计算图调度、数据管线可靠性、模型监控。
- 推荐策略：Petri网建模数据流与资源占用；动态图监控依赖与调用链；拓扑检测分布漂移/异常形状。
- 产出物：`03-AI基础设施应用模式/` 决策树+案例（训练/推理流水线、特征平台、A/B监控）。
