# 完整主题结构梳理与文件夹组织方案 / Complete Topic Structure and Folder Organization Plan 2025

## 📋 **用户需求 / User Requirements**

根据用户反馈：

- ❌ 不应该将所有内容堆在一个文件内
- ✅ 应该按照主题和子主题创建对应的文件夹和文件
- ✅ 便于后续扩展和维护

---

## 🎯 **组织原则 / Organization Principles**

### 1. 层次化文件夹结构

每个模块按照以下层次组织：

```
模块/
├── 00-模块元模型.md          # 元模型（保留）
├── 01-基础概念/              # 基础主题文件夹
│   ├── README.md            # 目录说明
│   ├── 01-子主题A.md
│   └── 02-子主题B.md
├── 02-核心理论/              # 核心理论文件夹
│   ├── README.md
│   ├── 01-理论A/
│   └── 02-理论B/
├── 03-高级理论/              # 高级理论文件夹
│   ├── README.md
│   ├── 01-高级主题A/
│   └── 02-高级主题B/
└── 99-应用案例/              # 应用案例文件夹
    └── ...
```

### 2. 文件夹命名规范

- **序号前缀**：`NN-` 格式，便于排序
- **中文名称**：清晰描述主题
- **英文对照**：在README中提供

### 3. 文件命名规范

- **序号前缀**：`NN-` 格式
- **简短名称**：准确描述内容
- **避免过长**：文件名不超过50个字符
