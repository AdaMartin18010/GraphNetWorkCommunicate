## 💻 **四、认知表征生成算法实现 / Part 4: Cognitive Representation Generation Algorithm Implementation**

### 4.1 思维导图生成算法

```python
class MindMapGenerator:
    """思维导图生成器"""

    def generate_mind_map(self, knowledge_structure):
        """
        从知识结构生成思维导图

        知识结构：
        - 主题
        - 子主题
        - 概念
        - 关系
        """
        mind_map = {
            'root': knowledge_structure['topic'],
            'branches': []
        }

        # 为每个子主题创建分支
        for subtopic in knowledge_structure.get('subtopics', []):
            branch = {
                'subtopic': subtopic['name'],
                'concepts': subtopic.get('concepts', []),
                'children': []
            }

            # 递归处理子分支
            if 'children' in subtopic:
                branch['children'] = self._generate_branches(subtopic['children'])

            mind_map['branches'].append(branch)

        return mind_map

    def _generate_branches(self, children):
        """递归生成分支"""
        branches = []
        for child in children:
            branch = {
                'name': child['name'],
                'concepts': child.get('concepts', []),
                'children': []
            }
            if 'children' in child:
                branch['children'] = self._generate_branches(child['children'])
            branches.append(branch)
        return branches
```

### 4.2 关系图构建算法

```python
class RelationshipGraphBuilder:
    """关系图构建器"""

    def build_relationship_graph(self, concepts, relationships):
        """
        构建概念关系网络图

        输入：
        - concepts: 概念列表
        - relationships: 关系列表（源概念、目标概念、关系类型）
        """
        graph = nx.DiGraph()

        # 添加节点（概念）
        for concept in concepts:
            graph.add_node(concept['id'], **concept)

        # 添加边（关系）
        for rel in relationships:
            graph.add_edge(
                rel['source'],
                rel['target'],
                relation_type=rel['type'],
                weight=rel.get('weight', 1.0)
            )

        return graph

    def analyze_centrality(self, graph):
        """分析关系图的中心性"""
        centrality = {
            'degree': nx.degree_centrality(graph),
            'betweenness': nx.betweenness_centrality(graph),
            'closeness': nx.closeness_centrality(graph),
            'pagerank': nx.pagerank(graph)
        }
        return centrality
```

### 4.3 因果链分析算法

```python
class CausalChainAnalyzer:
    """因果链分析器"""

    def build_causal_chain(self, events, causal_relations):
        """
        构建因果链

        输入：
        - events: 事件列表
        - causal_relations: 因果关系列表（原因事件、结果事件、强度）
        """
        chain = {
            'events': events,
            'causal_relations': causal_relations,
            'paths': []
        }

        # 构建因果图
        causal_graph = nx.DiGraph()
        for event in events:
            causal_graph.add_node(event['id'], **event)

        for rel in causal_relations:
            causal_graph.add_edge(
                rel['cause'],
                rel['effect'],
                strength=rel.get('strength', 1.0)
            )

        # 找到所有因果路径
        for source in events:
            for target in events:
                if source['id'] != target['id']:
                    paths = list(nx.all_simple_paths(
                        causal_graph, source['id'], target['id']
                    ))
                    if paths:
                        chain['paths'].extend(paths)

        return chain
```

## 📊 **五、复杂度分析与性能 / Part 5: Complexity Analysis and Performance**

### 时间复杂度

**思维导图生成**:

- 知识结构解析: $O(n)$ 其中 $n$ 是概念数
- 分支生成: $O(n)$
- **总复杂度**: $O(n)$

**关系图构建**:

- 节点添加: $O(n)$
- 边添加: $O(m)$ 其中 $m$ 是关系数
- 中心性分析: $O(n^2)$ 或 $O(n \cdot m)$
- **总复杂度**: $O(n^2 + m)$

**因果链分析**:

- 因果图构建: $O(n + m)$
- 路径查找: $O(n!)$（最坏情况）
- **总复杂度**: $O(n!)$（实际中通常更小）

### 空间复杂度

- **思维导图生成**: $O(n)$
- **关系图构建**: $O(n + m)$
- **因果链分析**: $O(n + m)$
