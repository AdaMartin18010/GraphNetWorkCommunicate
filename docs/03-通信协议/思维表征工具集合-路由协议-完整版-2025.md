## 📊 **二、概念多维矩阵 / Multi-dimensional Concept Matrices**

### 2.1 路由协议对比矩阵

| 维度 | RIP | OSPF | BGP | EIGRP |
|------|-----|------|-----|-------|
| **协议类型** | 距离向量 | 链路状态 | 路径向量 | 高级距离向量 |
| **更新方式** | 定期广播 | 事件触发泛洪 | 增量更新 | 部分更新 |
| **收敛速度** | 慢 | 快 | 中等 | 快 |
| **可扩展性** | 低(最大15跳) | 高(支持区域) | 极高(Internet规模) | 中等 |
| **复杂度** | 低 | 高 | 极高 | 中等 |
| **适用场景** | 小型网络 | 中型企业网络 | 大型ISP网络 | 企业网络 |
| **路由度量** | 跳数 | 成本 | 路径属性 | 复合度量 |
| **优缺点** | 简单但收敛慢 | 高效但复杂 | 可扩展但复杂 | 快速但专有 |
| **最新优化** | 触发更新 | 快速收敛 | BGPsec安全 | 多协议支持 |

### 2.2 路由算法对比矩阵

| 维度 | Dijkstra算法 | Bellman-Ford算法 | SPF算法 | 路由协议算法 |
|------|-------------|----------------|---------|------------|
| **定义** | 单源最短路径算法 | 单源最短路径算法 | 最短路径优先算法 | 路由协议使用的算法 |
| **关系** | 静态算法 | 动态算法 | 链路状态协议基础 | 实际应用算法 |
| **时间复杂度** | O(V²)或O(E log V) | O(VE) | O(V²) | 协议相关 |
| **适用场景** | 稠密图、静态网络 | 有负权边 | 链路状态协议 | 实际网络 |
| **优缺点** | 快速但需完整图信息 | 支持负权但慢 | 快速但需完整信息 | 实用但复杂度高 |
| **最新优化** | 堆优化、并行化 | 增量更新 | 区域化 | AI优化 |

## 🌲 **四、证明树图 / Proof Trees**

### 4.1 Dijkstra算法正确性证明树

```mermaid
graph TD
    Theorem[Dijkstra算法正确性<br/>找到单源最短路径] --> Initialize[初始化:<br/>距离d[s]=0<br/>其他d[v]=∞]

    Initialize --> Iterate[迭代过程:<br/>选择未访问的最近节点u<br/>更新u的邻居节点距离]

    Iterate --> Greedy[贪心选择:<br/>每次选择最近的节点<br/>保证局部最优]

    Greedy --> Update[距离更新:<br/>d[v] = min(d[v], d[u] + w(u,v))<br/>松弛操作]

    Update --> Optimal[最优性:<br/>已访问节点的距离<br/>是最短距离]

    Optimal --> Inductive[归纳证明:<br/>假设k个节点已正确<br/>第k+1个节点也正确]

    Inductive --> Correctness[正确性:<br/>所有节点距离<br/>都是最短距离]

    Correctness --> Conclusion[结论: Dijkstra算法<br/>正确找到最短路径]

    style Theorem fill:#e1f5ff
    style Conclusion fill:#d4edda
    style Optimal fill:#fff3cd
```

### 4.2 距离向量协议收敛性证明树

```mermaid
graph TD
    Theorem[距离向量协议收敛性<br/>在无环路网络中收敛到最短路径] --> Assumption[假设: 网络无环路<br/>链路成本为正]

    Assumption --> Bellman[Bellman方程:<br/>d[v] = min(d[u] + w(u,v))<br/>对所有邻居u]

    Bellman --> Iteration[迭代更新:<br/>每个节点向邻居<br/>发送距离向量]

    Iteration --> Convergence[收敛性:<br/>每次迭代距离单调递减<br/>且下界为最短距离]

    Convergence --> Finite[有限性:<br/>最多V次迭代<br/>必定收敛]

    Finite --> Correctness[正确性:<br/>收敛到最短路径<br/>满足Bellman方程]

    Correctness --> Conclusion[结论: 距离向量协议<br/>在无环路网络中收敛]

    style Theorem fill:#e1f5ff
    style Conclusion fill:#d4edda
    style Convergence fill:#fff3cd
```

## 🔄 **五、控制执行数据流图 / Control Flow & Data Flow Diagrams**

### 5.1 RIP协议更新流程

```mermaid
flowchart TD
    Start([开始RIP更新]) --> Timer[定时器触发<br/>每30秒或链路变化]

    Timer --> Build[构建路由表更新<br/>包含所有已知路由]

    Build --> Broadcast[向所有邻居<br/>广播路由更新]

    Broadcast --> Receive[接收邻居<br/>路由更新]

    Receive --> Process[处理更新:<br/>检查路由条目]

    Process --> Check{路由是否<br/>已知?}

    Check -->|新路由| Add[添加新路由<br/>距离+1]
    Check -->|已知路由| Compare{新距离<br/>< 当前距离?}

    Compare -->|是| Update[更新路由表<br/>更新距离和下一跳]
    Compare -->|否| Keep[保持当前路由]

    Add --> Trigger{是否触发<br/>更新?}
    Update --> Trigger
    Keep --> Wait[等待下次更新]

    Trigger -->|是| Timer
    Trigger -->|否| Wait

    Wait --> End([RIP更新结束])

    style Start fill:#e1f5ff
    style End fill:#d4edda
    style Update fill:#fff3cd
```

### 5.2 OSPF协议LSA泛洪流程

```mermaid
flowchart TD
    Start([开始LSA泛洪]) --> Change[检测链路状态变化<br/>或定时更新]

    Change --> Create[创建LSA<br/>链路状态通告]

    Create --> Send[向所有邻居<br/>发送LSA]

    Send --> Receive[邻居接收LSA]

    Receive --> Check{是否已收到<br/>此LSA?}

    Check -->|是| Discard[丢弃LSA<br/>避免重复]
    Check -->|否| Process[处理LSA:<br/>更新LSDB]

    Process --> Flood[泛洪LSA:<br/>向其他邻居转发<br/>除来源邻居]

    Flood --> UpdateLSDB[更新链路状态数据库<br/>LSDB]

    UpdateLSDB --> Recompute[重新计算<br/>最短路径树]

    Recompute --> Update[更新路由表<br/>基于新SPT]

    Update --> End([LSA泛洪结束])

    Discard --> End

    style Start fill:#e1f5ff
    style End fill:#d4edda
    style Recompute fill:#fff3cd
```

### 5.3 BGP协议路径选择流程

```mermaid
flowchart TD
    Start([开始BGP路径选择]) --> Receive[接收BGP更新<br/>包含AS路径]

    Receive --> Parse[解析路径属性:<br/>AS路径、下一跳、<br/>MED、本地优先级]

    Parse --> Filter[应用入站策略:<br/>过滤路由<br/>修改属性]

    Filter --> BestPath{选择最佳路径<br/>基于路径选择算法}

    BestPath --> Rule1[规则1: 最高本地优先级]
    Rule1 --> Rule2[规则2: 最短AS路径]
    Rule2 --> Rule3[规则3: 最低来源类型]
    Rule3 --> Rule4[规则4: 最低MED值]
    Rule4 --> Rule5[规则5: 最短IGP路径]

    Rule5 --> Select[选择最佳路径<br/>作为活动路径]

    Select --> Install[安装到路由表<br/>或RIB-In]

    Install --> Outbound[应用出站策略:<br/>决定是否通告<br/>给其他邻居]

    Outbound --> Advertise{是否通告?}

    Advertise -->|是| Send[向邻居发送<br/>BGP更新]
    Advertise -->|否| Keep[仅本地使用]

    Send --> End([BGP路径选择结束])
    Keep --> End

    style Start fill:#e1f5ff
    style End fill:#d4edda
    style Select fill:#fff3cd
```
