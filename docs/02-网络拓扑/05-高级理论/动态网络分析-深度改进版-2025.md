## 🎯 **1. 动态网络的多种等价定义 / Multiple Equivalent Definitions**

动态网络有多种等价的定义方式，反映了不同的数学视角和计算需求。

### 1.1 时间序列定义（时间序列模型）

**定义 1.1.1** (动态网络 - 时间序列定义)

动态网络是图的时间序列，每个时间点对应一个网络快照。

**形式化表示**:

- 时间序列: $\mathcal{G} = (G_1, G_2, \ldots, G_T)$，其中 $G_t = (V_t, E_t)$ 是时刻 $t$ 的网络
- 快照序列: 每个 $G_t$ 是网络的快照
- 时间窗口: $T = \{1, 2, \ldots, T\}$ 是时间窗口

**特点**:

- 最直观的定义方式
- 强调时间序列
- 适合批量分析

### 1.2 事件序列定义（事件模型）

**定义 1.1.2** (动态网络 - 事件序列定义)

动态网络是节点和边的添加/删除事件序列。

**形式化表示**:

- 事件序列: $\mathcal{E} = (e_1, e_2, \ldots, e_m)$，其中 $e_i = (op, u, v, t_i)$
- 事件类型: $op \in \{add\_edge, delete\_edge, add\_node, delete\_node\}$
- 时间戳: $t_1 \leq t_2 \leq \ldots \leq t_m$

**特点**:

- 强调事件驱动
- 适合流式处理
- 便于实时分析

### 1.3 张量定义（张量模型）

**定义 1.1.3** (动态网络 - 张量定义)

动态网络是时序邻接张量，维度对应节点和时间。

**形式化表示**:

- 邻接张量: $A \in \{0,1\}^{n \times n \times T}$，其中 $A_{ijt} = 1$ 表示节点 $i$ 和 $j$ 在时刻 $t$ 连接
- 时间维度: $T$ 是时间维度
- 节点维度: $n$ 是节点数

**特点**:

- 强调张量结构
- 适合数学分析
- 便于计算

### 1.4 分析定义（分析模型）

**定义 1.1.4** (动态网络 - 分析定义)

动态网络分析是分析网络结构随时间演化的方法和工具。

**形式化表示**:

- 分析方法: $f: \mathcal{G} \to \mathcal{R}$，将动态网络映射到分析结果
- 分析结果: $\mathcal{R}$ 是分析结果（如演化模式、关键事件等）
- 分析工具: 包括路径分析、社区检测、中心性分析等

**特点**:

- 强调分析方法
- 适合实际应用
- 便于工具开发

### 1.5 范畴论定义（范畴模型）

**定义 1.1.5** (动态网络 - 范畴论定义)

动态网络是网络范畴 $\mathbf{Network}$ 中的时序函子，将时间映射到网络。

**形式化表示**:

- 时间范畴: $\mathbf{Time}$（对象为时间点，态射为时间顺序）
- 网络范畴: $\mathbf{Network}$（对象为网络，态射为网络变换）
- 时序函子: $F: \mathbf{Time} \to \mathbf{Network}$

**特点**:

- 抽象层次高
- 统一理论框架
- 便于与其他理论建立联系

## 🔬 **2. 核心性质与定理 / Core Properties and Theorems**

### 2.1 动态网络的基本性质

**性质 2.1.1** (时间窗口性质)

对于时间窗口 $[t_1, t_2]$，窗口网络 $G_{[t_1,t_2]} = \bigcup_{t=t_1}^{t_2} G_t$ 保持了原动态网络的某些结构性质。

**完整证明**:

**时间窗口定义**：

时间窗口 $[t_1, t_2]$ 的窗口网络是时间窗口内所有快照的并集：
$$G_{[t_1,t_2]} = \left(\bigcup_{t=t_1}^{t_2} V_t, \bigcup_{t=t_1}^{t_2} E_t\right)$$

**结构性质保持**：

**引理1**：如果动态网络的所有快照都满足某个结构性质（如连通性），则窗口网络也满足该性质。

**证明**：

如果所有快照 $G_t$（$t_1 \leq t \leq t_2$）都连通，则窗口网络 $G_{[t_1,t_2]}$ 也连通（因为窗口网络包含所有快照的边）。

**时间窗口性质**：

**定理**：对于时间窗口 $[t_1, t_2]$，窗口网络 $G_{[t_1,t_2]}$ 保持了原动态网络的某些结构性质。

**证明**：

由引理1，如果所有快照都满足某个结构性质，则窗口网络也满足该性质。

**结论**：对于时间窗口 $[t_1, t_2]$，窗口网络 $G_{[t_1,t_2]}$ 保持了原动态网络的某些结构性质。$\square$

**性质 2.1.2** (演化稳定性)

如果动态网络的演化是稳定的，则网络结构的变化是平滑的。

**完整证明**:

**演化稳定性定义**：

演化稳定性是指网络结构随时间平滑变化，不存在突然的结构突变。

**平滑演化**：

**引理1**：如果动态网络的演化是稳定的，则相邻时间点的网络结构相似。

**证明**：

如果演化稳定，则：

- 节点集合变化小：$|V_{t+1} \setminus V_t| + |V_t \setminus V_{t+1}|$ 小
- 边集合变化小：$|E_{t+1} \setminus E_t| + |E_t \setminus E_{t+1}|$ 小

因此相邻时间点的网络结构相似。

**演化稳定性**：

**定理**：如果动态网络的演化是稳定的，则网络结构的变化是平滑的。

**证明**：

由引理1，如果演化稳定，则相邻时间点的网络结构相似，网络结构的变化是平滑的。

**结论**：如果动态网络的演化是稳定的，则网络结构的变化是平滑的。$\square$

**性质 2.1.3** (时间可达性)

动态网络中的时间可达性是指存在时间尊重路径（边的时间戳单调递增）。

**完整证明**:

**时间尊重路径定义**：

时间尊重路径是路径 $P = (v_0, v_1, \ldots, v_k)$，使得边 $(v_i, v_{i+1})$ 的时间戳 $t_i$ 满足 $t_0 \leq t_1 \leq \ldots \leq t_{k-1}$。

**时间可达性**：

**引理1**：节点 $u$ 在时刻 $t$ 时间可达节点 $v$，当且仅当存在从 $u$ 到 $v$ 的时间尊重路径。

**证明**：

如果存在时间尊重路径从 $u$ 到 $v$，则 $u$ 在时刻 $t$ 时间可达 $v$。

如果 $u$ 在时刻 $t$ 时间可达 $v$，则存在时间尊重路径（由时间可达性的定义）。

**时间可达性**：

**定理**：动态网络中的时间可达性由时间尊重路径决定。

**证明**：

由引理1，时间可达性等价于存在时间尊重路径。

**结论**：动态网络中的时间可达性由时间尊重路径决定。$\square$

### 2.2 动态连通性定理

**定理 2.2.1** (动态连通性查询复杂度)

对于动态网络，使用动态并查集可以在 $O(\alpha(n))$ 时间内完成连通性查询。

**形式化表述**:

- 查询复杂度: $O(\alpha(n))$，其中 $\alpha(n)$ 是反阿克曼函数
- 更新复杂度: $O(\alpha(n))$（添加或删除边）

**完整证明**:

参见 [动态图理论-深度改进版-2025.md](../01-图论基础/05-高级理论/动态图理论-深度改进版-2025.md) 中的动态连通性查询复杂度定理（定理 2.2.1）。

## 🧠 **5. 算法与方法 / Algorithms and Methods**

### 5.1 时间窗口分析算法

**算法描述**:

时间窗口分析算法分析动态网络在时间窗口内的性质。

**算法步骤**:

1. 输入: 动态网络 $\mathcal{G} = (G_1, G_2, \ldots, G_T)$，时间窗口 $[t_1, t_2]$
2. 构建窗口网络: $G_{[t_1,t_2]} = \bigcup_{t=t_1}^{t_2} G_t$
3. 分析窗口网络: 分析窗口网络的结构性质（如连通性、中心性等）
4. 输出: 窗口网络的分析结果

**复杂度分析**:

- 时间复杂度: $O((t_2-t_1) \cdot m)$，其中 $m$ 是平均边数
- 空间复杂度: $O(n^2)$（存储窗口网络）

**正确性**:

- 时间窗口分析算法保证窗口网络构建正确
- 算法正确实现时间窗口分析

### 5.2 动态中心性算法

**算法描述**:

动态中心性算法计算节点在动态网络中的中心性。

**算法步骤**:

1. 输入: 动态网络 $\mathcal{G} = (G_1, G_2, \ldots, G_T)$
2. 时间聚合: 对于每个时间点 $t$，计算节点中心性 $c_i(t)$
3. 时间聚合: 聚合时间序列的中心性（如平均值、最大值等）
4. 输出: 动态中心性 $c_i$

**复杂度分析**:

- 时间复杂度: $O(T \cdot n^3)$（$T$ 个时间点，每个时间点 $O(n^3)$）
- 空间复杂度: $O(T \cdot n)$（存储时间序列的中心性）

**正确性**:

- 动态中心性算法保证中心性计算正确
- 算法正确实现动态中心性
