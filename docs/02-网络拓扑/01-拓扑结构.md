## 🎯 **1. 网络拓扑基本定义 / Basic Network Topology Definitions**

### 1.1 网络拓扑 / Network Topology

**定义 1.1** (网络拓扑 / Network Topology)
**网络拓扑**是网络节点和链路的几何排列方式，定义了网络中设备之间的物理或逻辑连接关系。

**形式化定义**：
$$T = (N, L, A)$$
其中：

- $N$ 是**节点集**（node set），$N \neq \emptyset$
- $L$ 是**链路集**（link set），$L \subseteq N \times N$
- $A$ 是**属性集**（attribute set），包含权重、容量、时延等

**历史背景**：

- **1960年代**：ARPANET开创网络拓扑研究
- **1970年代**：环形和星型拓扑广泛应用
- **1980年代**：以太网总线拓扑标准化
- **1990年代**：互联网层次化拓扑发展
- **2000年代**：小世界和无标度网络理论兴起

**应用领域**：

- **计算机网络**：局域网、广域网设计
- **电信网络**：电话网络、移动通信
- **数据中心**：服务器互连、存储网络
- **物联网**：传感器网络、智能家居

### 1.2 拓扑图 / Topology Graph

**定义 1.2** (拓扑图 / Topology Graph)
**拓扑图**是网络拓扑的图论表示，其中节点表示网络设备，边表示物理或逻辑连接。

**形式化定义**：
$$G_T = (V_T, E_T, w_T)$$
其中：

- $V_T$ 是顶点集，对应网络节点
- $E_T$ 是边集，对应网络链路
- $w_T: E_T \to \mathbb{R}^+$ 是权重函数

**性质**：

- 连通性：保证网络可达性
- 冗余性：提供故障恢复能力
- 可扩展性：支持网络增长
- 效率性：优化资源利用

## 🔗 **2. 基本拓扑结构 / Basic Topology Structures**

### 2.1 星型拓扑 / Star Topology

**定义 2.1** (星型拓扑 / Star Topology)
**星型拓扑**是中心节点与所有其他节点直接相连的网络结构。

**形式化定义**：
$$T_{star} = (N, L) \text{ where } N = \{c, n_1, n_2, \ldots, n_k\} \land L = \{(c, n_i) \mid i = 1, 2, \ldots, k\}$$

**拓扑图表示**：

- 中心节点度数为 $k$
- 叶子节点度数为 1
- 总边数：$|L| = k$

**性质**：

- **连通性**：任意两节点间最多2跳
- **容错性**：中心节点故障导致全网瘫痪
- **扩展性**：添加节点容易，但中心节点成为瓶颈
- **成本**：链路数量为 $O(n)$

**应用**：

- **以太网集线器**：早期局域网
- **无线网络**：WiFi接入点
- **电话网络**：PBX系统
- **传感器网络**：集中式数据收集

### 2.2 环形拓扑 / Ring Topology

**定义 2.2** (环形拓扑 / Ring Topology)
**环形拓扑**是节点按环形排列，每个节点与相邻节点相连的网络结构。

**形式化定义**：
$$T_{ring} = (N, L) \text{ where } N = \{n_1, n_2, \ldots, n_k\} \land L = \{(n_i, n_{i+1}) \mid i = 1, 2, \ldots, k-1\} \cup \{(n_k, n_1)\}$$

**拓扑图表示**：

- 每个节点度数为 2
- 总边数：$|L| = k$
- 图论性质：$C_k$（k阶圈图）

**性质**：

- **连通性**：任意两节点间最多 $\lfloor k/2 \rfloor$ 跳
- **容错性**：单点故障不影响其他节点通信
- **扩展性**：添加节点需要重新配置
- **成本**：链路数量为 $O(n)$

**应用**：

- **令牌环网络**：IBM Token Ring
- **光纤分布式数据接口**：FDDI
- **同步数字体系**：SDH/SONET环
- **区块链网络**：共识机制

### 2.3 总线拓扑 / Bus Topology

**定义 2.3** (总线拓扑 / Bus Topology)
**总线拓扑**是所有节点共享一条公共传输介质的网络结构。

**形式化定义**：
$$T_{bus} = (N, L) \text{ where } N = \{n_1, n_2, \ldots, n_k\} \land L = \{(n_i, n_j) \mid i \neq j\}$$

**拓扑图表示**：

- 完全图 $K_k$
- 每个节点度数为 $k-1$
- 总边数：$|L| = \frac{k(k-1)}{2}$

**性质**：

- **连通性**：任意两节点间1跳可达
- **容错性**：总线故障导致全网瘫痪
- **扩展性**：添加节点增加总线负载
- **成本**：链路数量为 $O(n^2)$

**应用**：

- **以太网总线**：早期10Base2/10Base5
- **CAN总线**：汽车网络
- **工业控制网络**：现场总线
- **内存总线**：计算机内部互连

### 2.4 树型拓扑 / Tree Topology

**定义 2.4** (树型拓扑 / Tree Topology)
**树型拓扑**是层次化网络结构，具有根节点、内部节点和叶子节点。

**形式化定义**：
$$T_{tree} = (N, L) \text{ where } N = \{r, I, L\} \land L \subseteq N \times N \land \text{acyclic}(T_{tree}) \land \text{connected}(T_{tree})$$

其中：

- $r$ 是根节点
- $I$ 是内部节点集
- $L$ 是叶子节点集

**拓扑图表示**：

- 无环连通图
- 任意两节点间有唯一路径
- 边数：$|L| = |N| - 1$

**性质**：

- **连通性**：任意两节点间有唯一路径
- **容错性**：单点故障分割网络
- **扩展性**：层次化扩展容易
- **成本**：链路数量为 $O(n)$

**应用**：

- **域名系统**：DNS层次结构
- **文件系统**：目录树结构
- **组织架构**：公司管理层次
- **网络路由**：路由表层次

### 2.5 网格拓扑 / Mesh Topology

**定义 2.5** (网格拓扑 / Mesh Topology)
**网格拓扑**是节点按规则网格排列的网络结构，每个节点与相邻网格节点相连。

**形式化定义**：
$$T_{mesh} = (N, L) \text{ where } N = \{n_{i,j} \mid 1 \leq i \leq m, 1 \leq j \leq n\}$$
$$L = \{(n_{i,j}, n_{i+1,j}) \mid 1 \leq i < m\} \cup \{(n_{i,j}, n_{i,j+1}) \mid 1 \leq j < n\}$$

**拓扑图表示**：

- $m \times n$ 网格
- 内部节点度数为 4
- 边界节点度数为 2 或 3
- 总边数：$|L| = m(n-1) + n(m-1) = 2mn - m - n$

**性质**：

- **连通性**：任意两节点间最多 $(m+n-2)$ 跳
- **容错性**：多条路径提供冗余
- **扩展性**：规则扩展容易
- **成本**：链路数量为 $O(mn)$

**应用**：

- **超级计算机**：处理器互连
- **传感器网络**：网格部署
- **无线网络**：WiFi网格
- **并行计算**：处理器阵列

## 🌐 **3. 高级拓扑结构 / Advanced Topology Structures**

### 3.1 小世界网络 / Small World Network

**定义 3.1** (小世界网络 / Small World Network)
**小世界网络**是具有高聚类系数和短平均路径长度的网络结构。

**形式化定义**：
$$T_{sw} = (N, L) \text{ where } C(T_{sw}) \gg C(T_{random}) \land L(T_{sw}) \approx L(T_{random})$$

其中：

- $C(T)$ 是聚类系数
- $L(T)$ 是平均路径长度

**Watts-Strogatz模型**：

1. 从规则环形网络开始
2. 以概率 $p$ 随机重连边
3. 当 $0 < p < 1$ 时产生小世界特性

**性质**：

- **高聚类**：局部结构紧密
- **短路径**：全局连接高效
- **随机性**：部分随机连接
- **鲁棒性**：对随机故障抵抗

**应用**：

- **社交网络**：朋友关系网络
- **神经网络**：大脑连接网络
- **电力网络**：电网拓扑
- **互联网**：AS级拓扑

### 3.2 无标度网络 / Scale-Free Network

**定义 3.2** (无标度网络 / Scale-Free Network)
**无标度网络**是节点度数分布遵循幂律分布的网络结构。

**形式化定义**：
$$P(k) \sim k^{-\gamma} \text{ where } \gamma > 0$$

其中：

- $P(k)$ 是度数为 $k$ 的节点概率
- $\gamma$ 是幂律指数

**Barabási-Albert模型**：

1. 从少量节点开始
2. 每次添加新节点
3. 新节点以偏好连接方式连接现有节点
4. 偏好概率与节点度数成正比

**性质**：

- **幂律分布**：少数节点度数很高
- **无标度性**：不同尺度下性质相似
- **鲁棒性**：对随机故障抵抗
- **脆弱性**：对目标攻击敏感

**应用**：

- **万维网**：网页链接网络
- **互联网**：路由器连接
- **蛋白质网络**：蛋白质相互作用
- **引用网络**：学术论文引用

### 3.3 层次化网络 / Hierarchical Network

**定义 3.3** (层次化网络 / Hierarchical Network)
**层次化网络**是具有明确层次结构的网络，不同层次具有不同的功能和特性。

**形式化定义**：
$$T_{hier} = (N, L, H) \text{ where } H: N \to \{1, 2, \ldots, h\}$$

其中：

- $H$ 是层次函数
- $h$ 是层次数

**层次结构**：

- **核心层**：高速骨干网络
- **汇聚层**：流量聚合和路由
- **接入层**：用户设备连接

**性质**：

- **层次性**：明确的功能分层
- **可扩展性**：模块化设计
- **可管理性**：分层管理
- **容错性**：层次化故障隔离

**应用**：

- **企业网络**：公司内部网络
- **数据中心**：服务器网络架构
- **电信网络**：运营商网络
- **互联网**：AS层次结构

## 📊 **4. 拓扑性质与度量 / Topology Properties and Metrics**

### 4.1 连通性度量 / Connectivity Metrics

**定义 4.1** (连通性 / Connectivity)
网络 $T$ 是**连通的**当且仅当任意两个节点间存在路径。

**形式化定义**：
$$\text{connected}(T) \iff \forall u, v \in N: \exists \text{ path } P: u \stackrel{P}{\to} v$$

**定义 4.2** (k-连通性 / k-Connectivity)
网络 $T$ 是**k-连通的**当且仅当移除任意 $k-1$ 个节点后仍连通。

**形式化定义**：
$$\text{k-connected}(T) \iff \forall S \subseteq N, |S| < k: \text{connected}(T - S)$$

**定义 4.3** (边连通性 / Edge Connectivity)
网络 $T$ 的**边连通性**是最小割的大小。

**形式化定义**：
$$\lambda(T) = \min_{S \subset N} |\delta(S)|$$

其中 $\delta(S)$ 是割集。

### 4.2 路径长度度量 / Path Length Metrics

**定义 4.4** (平均路径长度 / Average Path Length)
网络 $T$ 的**平均路径长度**是所有节点对间最短路径的平均值。

**形式化定义**：
$$L(T) = \frac{1}{n(n-1)} \sum_{u \neq v} d(u, v)$$

其中 $d(u, v)$ 是节点 $u$ 和 $v$ 间的最短路径长度。

**定义 4.5** (直径 / Diameter)
网络 $T$ 的**直径**是任意两节点间最短路径的最大值。

**形式化定义**：
$$D(T) = \max_{u, v \in N} d(u, v)$$

### 4.3 聚类度量 / Clustering Metrics

**定义 4.6** (聚类系数 / Clustering Coefficient)
节点 $v$ 的**聚类系数**是其邻居间实际边数与最大可能边数的比值。

**形式化定义**：
$$C_v = \frac{2|E_v|}{k_v(k_v-1)}$$

其中：

- $E_v$ 是节点 $v$ 邻居间的边集
- $k_v$ 是节点 $v$ 的度数

**定义 4.7** (全局聚类系数 / Global Clustering Coefficient)
网络的**全局聚类系数**是所有节点聚类系数的平均值。

**形式化定义**：
$$C(T) = \frac{1}{|N|} \sum_{v \in N} C_v$$

## 🔧 **5. 拓扑算法 / Topology Algorithms**

### 5.1 最短路径算法 / Shortest Path Algorithm

**算法 5.1** (Dijkstra算法 / Dijkstra's Algorithm)

```python
import heapq
from collections import defaultdict

def dijkstra(graph, start):
    """Dijkstra最短路径算法"""
    distances = defaultdict(lambda: float('inf'))
    distances[start] = 0
    pq = [(0, start)]
    visited = set()
    
    while pq:
        current_dist, current = heapq.heappop(pq)
        
        if current in visited:
            continue
            
        visited.add(current)
        
        for neighbor, weight in graph[current].items():
            distance = current_dist + weight
            
            if distance < distances[neighbor]:
                distances[neighbor] = distance
                heapq.heappush(pq, (distance, neighbor))
    
    return distances
```

### 5.2 连通性检测算法 / Connectivity Detection Algorithm

**算法 5.2** (连通分量检测 / Connected Components Detection)

```python
def find_connected_components(graph):
    """寻找连通分量"""
    visited = set()
    components = []
    
    def dfs(node, component):
        visited.add(node)
        component.add(node)
        for neighbor in graph[node]:
            if neighbor not in visited:
                dfs(neighbor, component)
    
    for node in graph:
        if node not in visited:
            component = set()
            dfs(node, component)
            components.append(component)
    
    return components
```

### 5.3 聚类系数计算算法 / Clustering Coefficient Algorithm

**算法 5.3** (聚类系数计算 / Clustering Coefficient Calculation)

```python
def clustering_coefficient(graph):
    """计算全局聚类系数"""
    total_coefficient = 0
    node_count = 0
    
    for node in graph:
        neighbors = list(graph[node])
        if len(neighbors) < 2:
            continue
            
        # 计算邻居间的边数
        neighbor_edges = 0
        for i in range(len(neighbors)):
            for j in range(i + 1, len(neighbors)):
                if neighbors[j] in graph[neighbors[i]]:
                    neighbor_edges += 1
        
        # 计算聚类系数
        max_edges = len(neighbors) * (len(neighbors) - 1) // 2
        if max_edges > 0:
            coefficient = neighbor_edges / max_edges
            total_coefficient += coefficient
            node_count += 1
    
    return total_coefficient / node_count if node_count > 0 else 0
```

## 📈 **6. 拓扑优化 / Topology Optimization**

### 6.1 最小生成树 / Minimum Spanning Tree

**定义 6.1** (最小生成树 / Minimum Spanning Tree)
网络 $T$ 的**最小生成树**是连接所有节点的最小权重树。

**形式化定义**：
$$T_{mst} = \arg\min_{T' \subseteq T, \text{spanning tree}} \sum_{e \in T'} w(e)$$

**Kruskal算法**：

```python
def kruskal_mst(graph):
    """Kruskal最小生成树算法"""
    edges = []
    for u in graph:
        for v, weight in graph[u].items():
            edges.append((weight, u, v))
    
    edges.sort()
    
    parent = {node: node for node in graph}
    
    def find(x):
        if parent[x] != x:
            parent[x] = find(parent[x])
        return parent[x]
    
    def union(x, y):
        parent[find(x)] = find(y)
    
    mst = []
    for weight, u, v in edges:
        if find(u) != find(v):
            union(u, v)
            mst.append((u, v, weight))
    
    return mst
```

### 6.2 网络流优化 / Network Flow Optimization

**定义 6.2** (最大流 / Maximum Flow)
网络 $T$ 中从源 $s$ 到汇 $t$ 的**最大流**是最大可能的流量。

**Ford-Fulkerson算法**：

```python
def ford_fulkerson(graph, source, sink):
    """Ford-Fulkerson最大流算法"""
    def bfs_path(graph, source, sink):
        queue = [(source, [source])]
        visited = set()
        
        while queue:
            current, path = queue.pop(0)
            if current == sink:
                return path
            
            for neighbor, capacity in graph[current].items():
                if neighbor not in visited and capacity > 0:
                    visited.add(neighbor)
                    queue.append((neighbor, path + [neighbor]))
        
        return None
    
    residual = {u: {v: cap for v, cap in neighbors.items()} 
                for u, neighbors in graph.items()}
    
    max_flow = 0
    
    while True:
        path = bfs_path(residual, source, sink)
        if not path:
            break
        
        # 找到路径上的最小容量
        min_capacity = float('inf')
        for i in range(len(path) - 1):
            u, v = path[i], path[i + 1]
            min_capacity = min(min_capacity, residual[u][v])
        
        # 更新残差网络
        for i in range(len(path) - 1):
            u, v = path[i], path[i + 1]
            residual[u][v] -= min_capacity
            residual[v][u] = residual[v].get(u, 0) + min_capacity
        
        max_flow += min_capacity
    
    return max_flow
```
