## 🌲 **四、证明树图 / Proof Trees**

### 4.1 归结原理正确性证明树

```mermaid
graph TD
    Theorem[归结原理正确性<br/>归结是完备的证明方法] --> Soundness[可靠性:<br/>如果归结得到空子句<br/>则原始公式不可满足]

    Soundness --> Resolution[归结规则:<br/>从子句C1和C2<br/>归结得到新子句C]

    Resolution --> Completeness[完备性:<br/>如果公式不可满足<br/>则存在归结证明]

    Completeness --> Refutation[反证法:<br/>证明φ不可满足<br/>等价于证明¬φ可满足]

    Refutation --> Derivation[归结推导:<br/>从初始子句集<br/>逐步归结到空子句]

    Derivation --> Empty[空子句:<br/>表示矛盾<br/>证明不可满足]

    Empty --> Conclusion[结论: 归结原理<br/>是完备的证明方法]

    style Theorem fill:#e1f5ff
    style Conclusion fill:#d4edda
    style Empty fill:#fff3cd
```

### 4.2 归纳证明正确性证明树

```mermaid
graph TD
    Theorem[数学归纳法正确性<br/>P(0) ∧ (∀k. P(k) → P(k+1)) → ∀n. P(n)] --> Base[基础情况:<br/>证明P(0)成立]

    Base --> Inductive[归纳步骤:<br/>假设P(k)成立<br/>证明P(k+1)成立]

    Inductive --> Assumption[归纳假设:<br/>假设P(k)对所有k成立]

    Assumption --> Step[归纳步骤证明:<br/>从P(k)推导P(k+1)]

    Step --> Universal[全称量化:<br/>对所有n, P(n)成立]

    Universal --> Conclusion[结论: 数学归纳法<br/>是正确的证明方法]

    style Theorem fill:#e1f5ff
    style Conclusion fill:#d4edda
    style Step fill:#fff3cd
```

## 🔄 **五、控制执行数据流图 / Control Flow & Data Flow Diagrams**

### 5.1 归结证明执行流程

```mermaid
flowchart TD
    Start([开始归结证明]) --> Input[输入: 子句集S<br/>要证明的公式φ]

    Input --> CNF[转换为CNF:<br/>将公式转换为<br/>合取范式]

    CNF --> Negate[取反:<br/>¬φ转换为<br/>子句集]

    Negate --> Resolve[归结操作:<br/>选择两个子句<br/>进行归结]

    Resolve --> NewClause[生成新子句:<br/>应用归结规则<br/>得到新子句C]

    NewClause --> Check{新子句<br/>是否为空?}

    Check -->|否| Add[添加到子句集<br/>S = S ∪ {C}]
    Check -->|是| Success[证明成功:<br/>原始公式不可满足<br/>¬φ不成立, φ成立]

    Add --> Select[选择新的子句对<br/>继续归结]

    Select --> Resolve

    Success --> End([归结证明结束])

    style Start fill:#e1f5ff
    style End fill:#d4edda
    style NewClause fill:#fff3cd
```

### 5.2 交互式证明执行流程

```mermaid
flowchart TD
    Start([开始交互式证明]) --> Input[输入: 要证明的定理<br/>Theorem: P → Q]

    Input --> State[当前状态:<br/>目标: 证明Q<br/>假设: P]

    State --> Tactic{选择证明策略}

    Tactic -->|直接证明| Apply[应用规则:<br/>从假设P<br/>直接推导Q]
    Tactic -->|分解| Decompose[分解目标:<br/>将复杂目标<br/>分解为子目标]
    Tactic -->|归纳| Induct[应用归纳:<br/>对结构进行归纳]
    Tactic -->|重写| Rewrite[应用重写:<br/>使用等式重写]

    Apply --> Check{目标<br/>是否完成?}
    Decompose --> SubGoals[生成子目标:<br/>证明子目标<br/>完成原目标]
    Induct --> Cases[生成归纳情况:<br/>基础情况和<br/>归纳步骤]
    Rewrite --> Simplify[简化目标:<br/>应用重写规则<br/>简化表达式]

    SubGoals --> State
    Cases --> State
    Simplify --> Check

    Check -->|未完成| State
    Check -->|完成| Success[证明完成:<br/>所有目标都证明<br/>定理成立]

    Success --> End([交互式证明结束])

    style Start fill:#e1f5ff
    style End fill:#d4edda
    style Tactic fill:#fff3cd
```

### 5.3 SAT求解流程

```mermaid
flowchart TD
    Start([开始SAT求解]) --> Input[输入: 布尔公式F<br/>CNF形式]

    Input --> Preprocess[预处理:<br/>简化公式<br/>单元传播]

    Preprocess --> Decide[决策:<br/>选择一个变量<br/>赋值true/false]

    Decide --> Propagate[传播:<br/>单元传播<br/>推导隐含赋值]

    Propagate --> Conflict{是否<br/>冲突?}

    Conflict -->|是| Analyze[冲突分析:<br/>分析冲突原因<br/>学习子句]

    Analyze --> Backtrack[回溯:<br/>撤销决策<br/>选择新赋值]

    Backtrack --> Decide

    Conflict -->|否| Complete{所有变量<br/>都已赋值?}

    Complete -->|否| Decide
    Complete -->|是| Satisfiable[可满足:<br/>找到满足赋值<br/>公式可满足]

    Satisfiable --> End([SAT求解结束])

    style Start fill:#e1f5ff
    style End fill:#d4edda
    style Conflict fill:#fff3cd
```
