# 主题分类标准

## 一、标准概述

本文档定义了图论-网络-通信项目的主题分类标准，用于规范所有项目文件的内容组织和主题归属。

## 二、主题分类体系

### 2.1 一级主题分类（4大维度）

项目采用**4大维度**的主题分类体系：

1. **理论基础维度** (Theoretical Foundation)
2. **应用实践维度** (Applied Practice)
3. **认知科学维度** (Cognitive Science)
4. **哲学思辨维度** (Philosophical Reflection)

### 2.2 二级主题分类

每个一级主题下包含多个二级主题，详见 `THEME_HIERARCHY_STRUCTURE.md`。

### 2.3 三级主题分类

每个二级主题下可进一步细分为三级主题，形成完整的主题层次树。

## 三、主题分类原则

### 3.1 分类原则

1. **唯一性原则**: 每个主题只能归属于一个一级维度
2. **层次性原则**: 主题分类必须遵循层次结构
3. **完整性原则**: 所有内容必须能够归类到某个主题
4. **一致性原则**: 相同类型的内容使用相同的分类标准

### 3.2 分类方法

#### 方法1: 按内容性质分类

- **理论基础**: 公理、定义、定理、证明
- **应用实践**: 具体应用案例、跨学科应用
- **认知科学**: 学习机制、认知过程、教学法
- **哲学思辨**: 本体论、认识论、文化批判

#### 方法2: 按文件来源分类

- **view00.md**: 主要归类为理论基础维度
- **view01.md**: 主要归类为应用实践维度
- **view02.md**: 主要归类为认知科学维度
- **view03.md**: 主要归类为哲学思辨维度

#### 方法3: 按项目结构分类

- **04-数学/**: 理论基础维度
- **09-应用/**: 应用实践维度
- **02-认识/**: 认知科学维度
- **01-哲科/**: 哲学思辨维度

## 五、主题标签规范

### 5.1 主题标签格式

主题标签采用以下格式：

```
[维度代码]-[二级主题代码]-[三级主题代码]
```

例如：

- `00-1.1.1` 表示模块00-项目简介与定位-项目目标
- `01-1.1.1` 表示模块01-基本概念-图的定义
- `02-1.2.3` 表示模块02-网络动力学-博弈动力学
- `03-1.3.1` 表示模块03-安全协议-加密协议
- `04-1.2.1` 表示模块04-一致性协议-CAP定理

### 5.2 模块代码

- **00**: 项目概述 (Project Overview)
- **01**: 图论基础 (Graph Theory Foundations)
- **02**: 网络拓扑 (Network Topology)
- **03**: 通信协议 (Communication Protocols)
- **04**: 分布式系统 (Distributed Systems)
- **05**: 量子通信 (Quantum Communication)
- **06**: 生物网络 (Biological Networks)
- **07**: 社会网络 (Social Networks)
- **08**: 形式化证明 (Formal Verification)
- **09**: AI网络与自适应范畴 (AI Networks & Adaptive Categories)
- **10**: Petri网理论 (Petri Net Theory)
- **11**: 复杂系统与多尺度建模 (Complex Systems & Multi-scale Modeling)

### 5.3 主题标签使用

每个项目文件应在文件头部添加主题标签，格式如下：

```markdown
---
主题标签: [模块代码]-[一级主题编号].[二级主题编号].[三级主题编号]
模块代码: [模块代码]
一级主题: [主题名称]
二级主题: [主题名称]
三级主题: [主题名称]
项目位置: [项目路径]
---
```

## 八、主题分类维护

### 8.1 新增主题

当需要新增主题时：

1. 确定主题所属的核心模块
2. 确定主题所属的一级主题
3. 确定主题所属的二级主题
4. 确定主题所属的三级主题（如适用）
5. 更新 `THEME_HIERARCHY_STRUCTURE.md`
6. 更新相关文件的主题标签

### 8.2 主题调整

当需要调整主题分类时：

1. 评估调整的必要性
2. 更新 `THEME_HIERARCHY_STRUCTURE.md`
3. 更新相关文件的主题标签
4. 更新项目结构映射关系

### 8.3 主题合并

当需要合并主题时：

1. 评估合并的合理性
2. 更新 `THEME_HIERARCHY_STRUCTURE.md`
3. 合并相关文件内容
4. 更新主题标签
