## 🔧 批量标记方法

### 方法1: 使用脚本（推荐）

创建脚本 `mark-pending-files.sh`:

```bash
#!/bin/bash
# 为所有待完善文件添加标记

MARKER='⚠️ **状态**: 待完善
📝 **说明**: 本文档为深度改进版模板，内容不完整，需要根据项目定位补充完整的理论梳理内容。

**待补充内容**:
- [ ] 完整的理论定义
- [ ] 性质与定理
- [ ] 形式化证明
- [ ] 应用案例
- [ ] 与其他理论的关系

---'

find docs -name "*-深度改进版-2025.md" -type f | while read file; do
    # 检查是否已有标记
    if ! grep -q "⚠️ \*\*状态\*\*: 待完善" "$file"; then
        # 在标题后添加标记
        sed -i "1a\\n$MARKER\n" "$file"
        echo "已标记: $file"
    fi
done
```

### 方法2: 手动标记

对于重要文件，可以手动添加标记。标记应添加在文件标题之后，概述之前。

### 方法3: 使用编辑器批量替换

1. 使用支持批量查找替换的编辑器（如VS Code）
2. 查找模式: `## 📚 \*\*概述 / Overview\*\*`
3. 替换为: `⚠️ **状态**: 待完善\n📝 **说明**: 本文档为深度改进版模板，内容不完整，需要根据项目定位补充完整的理论梳理内容。\n\n**待补充内容**:\n- [ ] 完整的理论定义\n- [ ] 性质与定理\n- [ ] 形式化证明\n- [ ] 应用案例\n- [ ] 与其他理论的关系\n\n---\n\n## 📚 **概述 / Overview**`

## 🛠️ 批量标记脚本

已创建批量标记脚本，位于 `scripts/` 目录：

   ```powershell
   powershell -ExecutionPolicy Bypass -File scripts/mark-pending-files.ps1
   ```

2. **Bash脚本** (Linux/Mac): `scripts/mark-pending-files.sh`

   ```bash
   bash scripts/mark-pending-files.sh
   ```

**脚本功能**:

- 自动查找所有 `*-深度改进版-2025.md` 文件
- 检查是否已有标记
- 为未标记的文件添加待完善标记

## 🎯 标记优先级
